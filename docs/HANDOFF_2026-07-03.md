  all → on the mini, `pmpropose` errors every run, `pm_ideas.json` is never

  will fail** until the mini gets a PM route. This is open problem P1 (§4).

  Data still flows: build on MacBook → push to main → mini autopulls within
  5 min. Instant kickstarts need VPN off (then PM is off) — pick one per task.
- **The mini's autostash quirk**: `git pull --rebase --autostash` restores
  locally-dirty daemon-written files OVER pulled values. If a fresh committed
  data file doesn't take effect on the mini, run
  `git checkout origin/main -- <file>` there. Root fix = Phase-1 increment 2
  (untrack daemon artifacts), not yet executed.
- New launchd jobs NEVER self-activate: `bash deploy/macmini/install.sh` on
  the mini is a required human/SSH step after merging job definitions.
