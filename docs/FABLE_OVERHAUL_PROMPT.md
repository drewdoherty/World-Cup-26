### 2.6 DATA — event granularity (StatsBomb) vs market granularity (Polymarket CLOB)

**One-line thesis:** WCA holds the highest-resolution data available in BOTH its domains and reduces each to a low-dimensional summary before it ever reaches a model. **The football signal collapses to a final scoreline; the market signal collapses to a mid-price chart.** Fixing either closes a HAVE↔USE gap no amount of new data acquisition would.

- **martj42 results (`data/raw/martj42_cleaned.csv`):** ~49,498 matches (90-min scorelines) + `shootouts.csv` (runtime-downloaded, pins real pen winners for the bracket sim). **HIGH utilization** — the DC training corpus. Gap: dominated by low-scoring internationals, so its MLE intercept undershoots recent WC scoring by ~0.4–0.5 goals/match, patched by a **blunt scalar `level_target` shift** (`dixon_coles.py:653`).
- **StatsBomb events (`src/wca/data/statsbomb.py`, 428 lines):** ~128 WC-2018+2022 matches of **EVENT-LEVEL truth** — per-shot `statsbomb_xg`, on-target, per-player minutes, shot/SoT/xG/npxG shares, corners/cards/fouls with FIFA-correct second-yellow logic. **UTILIZATION ~0% IN PRODUCTION — DORMANT.** On the mini: `data/raw/statsbomb/` doesn't exist, `data/players.db` doesn't exist, no scheduled job builds it. The core DC model **never sees an xG number** — it fits on final scorelines only, which is *why* the scalar goal-level fudge exists. **Unlock:** ingest StatsBomb xG to (a) replace the scalar fudge with an empirical WC base rate, (b) shrink team attack/defence toward xG-derived priors (better signal than noisy goals-for), (c) resurrect player props on a real per-90 xG-share base.
- **odds_snapshots (prod `wca.db`):** **~3.24M rows**, 14 days, 100% `theoddsapi`, across **49 distinct bookmakers**, 70 matches, ~1,489 ticks — `h2h` 2.62M · `totals` 427k · `btts` 155k · `draw_no_bet` 37k. Adaptive cadence (dense in-game, sparse pre-game). **MEDIUM utilization:** totals/btts ARE now captured and consumed by `accas.py` + `arb.py` + settlement — but **the core goal model never consumes market totals as a signal.** The card prices O/U 2.5 and BTTS from its OWN DC lambdas (`card.py:1885`) and compares to book odds for EV; it never devigs the 427k market-totals rows back into goal expectancy. The 49-book cross-section is flattened to one devigged h2h number (no sharp-book/Pinnacle weighting, no per-book bias model); the dense h2h trajectory feeds CLV capture but no steam/movement model. **Unlock:** devig the totals surface into an empirical λ-level prior (replacing the scalar fudge), weight books by sharpness, mine the intra-match trajectory for steam.

The microstructure scripts themselves annotate "NO order-book depth, NO matched volume, NO queue position" — describing exactly the data the CLOB endpoints DO expose but the pipeline never retains. **Unlock:** a dense CLOB capture daemon (persist `prices-history` at 1-min + periodic top-of-book into a real tick table) enables momentum/steam models, mark-to-market at transactable bid-side exit prices, advancement term-structure, and realized-vol/order-flow edge that is impossible today because the data is discarded on arrival. The `wca_pm_analytics_suite.py` scaffolding (calibration, term structure, MTM) exists but runs ad-hoc only (in no `com.wca.*` service).
- **Promos/boosts:** `promotions` 37 · `promo_snapshots` 696 · **`boost_evals` 0** · `news_items` 30,421. Scraped and surfaced (`promosd`/`newsd` live) but the boost-EV loop has **written zero evaluations** — the `argmax(model_EV + promo − gub)` routing isn't persisting. **Two headline dormancies:** (1) StatsBomb event pipeline runs nowhere and the model has no xG anchor; (2) the Polymarket CLOB — the single richest live source — is captured at chart resolution with its order book discarded.

### 2.7 BETFAIR — can we trade, what's missing, is it worth it? (DECISIVE ANSWER)

**No — you cannot place a live Betfair order today, and building it is NOT worth it for this project's football-futures/exotics focus.** Betfair is wired as a **deliberately read-only / monitoring-only** source; execution is a hard-stopped stub.

- **What exists (2 modules, both read-only):** `src/wca/data/betfair.py` — thin Odds-API filter with an explicit `betfair_execution_stub()` (`:71`) that `raise NotImplementedError("read-only ... Betfair bet placement is intentionally unimplemented (monitoring-only). Use Polymarket ClobTrader for the only supported execution path.")`. `src/wca/data/betfair_exchange.py` (727 lines) — the real JSON-RPC client, but **only READ methods** (`listMarketCatalogue`, `listMarketBook`/`get_odds`, `listCurrentOrders`, `listClearedOrders`). Hard grep for every placement primitive (`placeOrders`/`cancelOrders`/`replaceOrders`/`PlaceInstruction`/`limitOrder`/`persistenceType`) across all Betfair files = **zero hits**. The only `place_order` in the codebase is Polymarket's.
- **Integration state:** wired first in `odds_source.py:28` priority ("Betfair wins the moment creds are present") but OFF — returns empty frame when creds missing. No Betfair-specific launchd service.
- **API access is configured-but-non-functional:** the mini's real `.env` HAS `BETFAIR_*` set — but a **delayed** key only (`BETFAIR_APP_KEY_DELAY`, no `_LIVE`), a manually-pasted session token that expires, **no client cert**, no session cache. And critically **the mini physically cannot reach Betfair** — `api.betfair.com`/`identitysso.betfair.com` fail with `SSL: WRONG_VERSION_NUMBER` (geo-block/interception) while general HTTPS works; the code comments already say the real Betfair fetch must run on the MacBook over VPN. So even the read feed is dead on the box that runs the bots.
- **Missing to reach live placement (all absent):** (1) an entire execution client from scratch (`placeOrders`/`cancelOrders`/`replaceOrders` + `customerRef` idempotency + a guardrail/sizing path mirroring `pm/trader.py` + ledger writes + settlement hooks); (2) provisioned creds — a 2048-bit RSA cert uploaded + cert-login wired; (3) a **~£499 one-off non-refundable Live App Key activation** + KYC + a delayed test bet during dev + a 1–3 business-day manual review; (4) sizing/routing wiring to convert the ¼-Kelly £3k rule into GBP Betfair stakes; (5) the 2026 WC **Passive Bet Delay** and the new Expert Fee to design around. Commission: code hardcodes `DEFAULT_BETFAIR_COMMISSION = 0.06` (6%) — *higher* than any real tier; Betfair now defaults to Rewards 5%, Basic 2% by opt-in.
- **Verdict for the overhaul:** **Keep `betfair_exchange.py` as the read-only CLV/closing-line reference it was designed to be** (the cheap £0 win is fixing the mini's network path or relaying via the MacBook/VPN). **Do NOT build Betfair execution.** Polymarket already has a complete, guardrailed, live-capable execution stack the whole sizing rule and fire button are built around; Betfair's only unique edge would be exotics PM lacks (Correct Score, Asian Handicap, in-play) — but the project's own research already **killed correct-score/scorer/SGM as −EV leaks**, and Betfair's Match-Odds/CS/O-U are 90-minute-only (a settlement mismatch that has bitten this project). **If a GBP exchange execution venue is ever wanted, Smarkets is the better first target** (2%, 0% on outrights, no ~£499 gate, no cert dance, already partially wired, has its own `smarkets_execution_stub` ready to fill).
