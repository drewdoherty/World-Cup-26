### 1.2 Method

Each unmerged remote branch was scored with four read-only measurements against `origin/main` (957112a):

1. **Merged check** — `git branch -r --merged origin/main`: tip is an ancestor of main → PRUNE, zero risk.
2. **Cherry-equivalence** — `git cherry origin/main <tip>`: a commit marked `-` has a patch-identical equivalent already on main (typically via squash-merge). 100% `-` (or ahead=0) → PRUNE-SUPERSEDED: the branch guards no unique diff.
3. **Ahead/behind** — `git rev-list --left-right --count origin/main...<tip>`: behind-count measures how stale a merge would be (588 behind = unmergeable in practice).
4. **Shortstat + name-only diff** — `git diff --shortstat/--name-only origin/main...<tip>`: sizes the unique diff and detects data-artifact churn mixed into code branches.

Everything else lands in REVIEW: the branch contains at least one commit with no patch-equivalent on main, so a human (or agent with a human gate) must look before deletion.

### 1.3 The shape of the problem

47 of 98 remote branches (48%) are deletable at near-zero risk today. The 51 REVIEW branches skew heavily toward Jun-21–25 conductor/codex one-shot experiments sitting 280–588 commits behind main — most will be judged obsolete on inspection, but each contains at least one non-equivalent commit so none is deleted blind. Single sanctioned root going forward: **`<repo>/worktrees/`**, added to `.gitignore` (today it is merely untracked — the `?? worktrees/` noise in `git status`). Everything else is removed.

## 7. Hygiene going forward (proposal)

1. **One worktree root.** All worktrees live under `<repo>/worktrees/` (gitignored). Session scratchpads and other projects' directories are never worktree roots — any agent tooling that creates worktrees (conductor, workflow scripts) receives the sanctioned root explicitly. `/private/tmp` roots are banned outright: macOS deletes them on reboot, stranding registrations and (twice in this audit) unique work.
2. **Branch naming.** `feat/ | fix/ | chore/ | ops/ | docs/<kebab-slug>`; conductor one-shots keep the `conductor/` prefix so the monthly prune can treat them as expire-by-default after 14 days unmerged.
3. **Delete-on-merge.** Enable GitHub "Automatically delete head branches". 41 of the 47 Tier-A branches exist only because squash-merged PR heads were never deleted — this single setting prevents the entire class from re-accumulating.
4. **Worktree lifecycle = PR lifecycle.** A worktree is created for a branch, and `git worktree remove`d when the branch's PR merges or closes. No parked worktrees.
5. **Monthly prune ritual** (~10 minutes, proposal): re-run `branch_audit.py`; delete every merged and every 100%-cherry-equivalent branch older than 7 days (local and remote); `git worktree prune`; eyeball anything unmerged older than 30 days; record counts in `docs/overhaul/prune_log.md`. Steady-state targets: <15 remote branches, <5 worktrees, zero worktrees outside the sanctioned root.
6. **No blind merges of old branches, ever.** The standing rule stays: cherry-pick code, never merge a stale branch (data regression risk); anything >100 behind is presumed extract-or-delete.
