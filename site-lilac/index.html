WC'26 - model probabilities vs live prices across best 19 Sportsbooks and Polymarket: Edges, sizing, exposure, P&L.

■ Design rationale

A real-money betting terminal lives and dies on glance-value: you need exposure, edge and best-price in one eyeful before a line moves. The wide 3-rail layout keeps venue/portfolio state pinned left, the live decision surface (predictions + positions) fluid in the centre, and time-sensitive alerts docked right — so the eye never scrolls to find risk. Master-detail on positions and fixtures means the list stays scannable while one selected bet expands into its full detail — depth on demand, not a wall of nested tables. Every figure on this build is live from the World Cup Alpha repo JSON (consolidated —); £ and $ are reported in separate cells and never blended or FX-converted, null model fields render as —, and stale/empty sources are surfaced honestly rather than papered over.

Execution Engine // Architecture Spec

spec
1 · Signal layer
Blended model (10% Elo + 30% Dixon-Coles + 60% market) emits fair probabilities; de-vigged consensus built per market.
2 · Pricing layer
Cross-venue snapshots (multi-book, Betfair Exchange, Polymarket) normalised to a common outcome schema; fees & FX disclosed per leg.
3 · Edge / sizing layer
Fee-adjusted EV per side; fractional-Kelly stake on separate GBP / USD pools; per-bet cap & whole-book hard-floor exposure.
4 · Routing layer
argmax(model EV + promo − gub-risk); default to exchange / Smarkets for lowest overround; account routing A1 / A2.
5 · Execution layer
SHADOW → DRY-ARM → LIVE safety ladder. Currently monitoring-only — no automated fills placed.
6 · Settlement / CLV layer
Closing-odds + CLV stamped at kickoff; uniform execution-CLV convention across all settle paths; prediction ledger persisted.
#1 architectural gap: odds capture stops pre-kickoff, so there is no true closing line — CLV is execution-relative, not market-close-relative. Two-sided live price feeds for props remain unwired.
Static · engine architecture spec
